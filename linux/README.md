## Linux学習記録

## 今日の学習：ディレクトリ移動の罠

### 失敗した操作
カレントディレクトリが `/home/user01` のとき、ルート直下の `/subdir1` に行こうとして以下を実行。
`$ cd subdir1/dir001`
→ **エラー：** `/home/user01/subdir1/dir001` を探しに行ってしまい、見つからない。

### 学んだこと
- パスの先頭に `/` がないと、**「今の場所」**から探し始める（相対パス）。
- どこからでも特定の場所へ行きたいときは、先頭に `/` をつけて**「ルート」**から指定する（絶対パス）。

### 正解のコマンド
- `$ cd /subdir1/dir001` （一発で移動）
- `$ cd ../../subdir1/dir001` （親ディレクトリを遡って移動）
