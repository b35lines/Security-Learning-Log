# 🛡️ 情報セキュリティマネジメント 攻略メモ

今日学習した「テキスト外の重要用語」や「混乱しやすい概念」のまとめ。

## 1. ネットワーク防御の三銃士
「どこで」「何を」守るかの違いが重要。

| 用語 | 守備範囲 | 主な役割 | イメージ |
| :--- | :--- | :--- | :--- |
| **FW (ファイアウォール)** | ネットワーク境界 | IPアドレスとポートで通していい通信か判断する。 | 玄関の門番 🚪 |
| **IPS (侵入防止システム)** | ネットワーク全体 | 通信の中身をリアルタイム監視し、攻撃を遮断する。 | 館内パトロール 👮 |
| **WAF** | Webアプリ | SQLインジェクション等、Web特有の攻撃を深くチェック。 | 受付の専用検査官 🌐 |

---

## 2. 攻撃と対策のキーワード
最新トレンドや「おとり」の技術。

- **サプライチェーン攻撃** 📦
    - 信頼している取引先やソフトウェアの更新ルートを悪用して侵入する手法。
- **ハニーポット** 🍯
    - あえて脆弱性を見せて攻撃者を誘い込み、手法を観察するための「おとり」システム。
- **レジリエンス** 🌱
    - 攻撃を100%防ぐのではなく、被害を受けても「素早く回復する」という考え方。

---

## 3. 組織・管理のルール
「人」による不正やミスを防ぐ仕組み。

- **職務分掌 (SoD)** 👥
    - 「申請」と「承認」のように役割を分け、一人で不正ができないようにすること。
- **最小権限の原則** 🔑
    - 業務に必要な「最低限」の権限だけを与えること（不要な権限は持たせない）。
- **プロキシサーバー** 代理
    - 社内から外へのWebアクセスを中継し、ログ記録や有害サイト遮断を行う。

---

## 💡 迷ったときのチェックリスト
- 外から中への攻撃を防ぐなら？ → **FW / IPS / WAF**
- 中から外への通信を管理するなら？ → **プロキシサーバー**
- 一人による不正を防ぐなら？ → **職務分掌**

## 🔍 ログと構成の可視化 (SIEM & SBOM)
「何が起きているか」「何でできているか」を把握するための技術。

| 用語 | 読み方 | 役割 | 導入するメリット |
| :--- | :--- | :--- | :--- |
| **SIEM** | シーム | 各種機器のログを集約・相関分析する。 | 単独のログでは気づけない「攻撃の流れ」を検知できる。 |
| **SBOM** | エスボム | ソフトウェアに含まれる部品（ライブラリ等）の一覧表。 | 脆弱性が見つかった際、自社ソフトに影響があるか即座に判断できる。 |

---

### 🛠️ SIEM (Security Information and Event Management) 🚨
- **仕組み**: FW、サーバー、PCなどのバラバラなログを1箇所に集めて、リアルタイムで分析する。
- **なぜ必要か**: 
    - 攻撃者は巧妙に侵入するため、一つの機器のログだけでは異常に見えないことが多い。
    - 「深夜に普段使わない端末が(PCログ)」「大量のデータを外部へ送信した(FWログ)」といった**点と点を結んで**、攻撃をあぶり出す。
- **イメージ**: 建物中の防犯カメラとセンサーを一括管理する「中央監視室」。

### 🛠️ SBOM (Software Bill of Materials) 📋
- **仕組み**: ソフトウェアの「成分表示表」や「お品書き」。
- **なぜ必要か**: 
    - 現代のソフトは多くの「オープンソース（外部部品）」を組み合わせて作られている。
    - 外部部品に欠陥（脆弱性）が見つかったとき、SBOMがないと「自社のどのソフトが危険か」を調べるだけで数週間かかってしまう。
- **イメージ**: 食品の「原材料ラベル」。アレルギー物質（脆弱性）が含まれていないか一目でわかる。
